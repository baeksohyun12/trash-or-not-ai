# 사용자 경험(UX)란 뭘까

기술블로그 링크 : [사용자 경험(UX)란 뭘까..](https://velog.io/@baeksohyun12/디자인사용자-경험UX란-뭘까)

### 기존 방식

“버릴까말까” 프로젝트 이전까지는 디자이너가 따로 없어 모든 파트(기획, 백엔드, 프론트엔드)가 함께 디자인을 고려하였다. 그러다 보니 어떻게 해야 좀 예쁘게 화면이 나올까 이런걸 많이 고민했었다.

### 문제 상황

실제로 디자인 파트를 맡아보니, 화면이 예쁘게 나오는 것보다 사용자의 입장에서 화면을 어떻게 구성해야 할지 고민하게 되었다. 와이어프레임부터 실제 디자인까지 다 맡아서 하다보니 막막하기도 하고 내 입장에서만 생각하게 되면서 가독성이 떨어지거나 하나의 결과를 얻기 위해 클릭하는 버튼의 수도 많아졌다. 

그러면서 자연스럽게 UI/UX에서 UX, 사용자 경험이 과연 뭔지 궁금해졌다. 

### 해결 방안

이번에도 역시 디자이너들의 기술 블로그를 참고하였다. 

우선, 사용자 경험의 정의는 “디지털 제품을 디자인하고 개발함으로써 사용자에게 원활하고 즐거운 경험을 제공하는 것”이다. 여기에는 디자인, 기능, 브랜딩 등 모든 요소를 고려하여 리서치, 프로토타이핑, 테스팅 등의 기법으로 사용자의 니즈와 행동을 이해하고 제품을 최적화하는 것을 목표로 한다. 따라서 사용자 경험 디자인을 이해하는 것은 사용자의 요구와 기대를 충족하는 잘 설계된 서비스를 만드는 데에 굉장히 중요한 역할을 한다는 것을 알 수 있었다.

이를 반영하기 위해 페르소나를 생성하고 페르소나의 입장에서 요구사항을 이해하고 화면 구성을 개선하였다. 또한 피그마의 프로토타입 기능을 활용하며 화면 흐름이 괜찮은지 생각하며 디자인 하였다. 

### 결과

전체적인 화면에 통일감을 주고 하나의 결과를 반환하는데까지 누르는 버튼의 수를 최소화할 수 있도록 하였다. 또한 다양한 연령대가 쉽게 사용할 수 있게 최대한 직관적으로 화면을 구성하고 아이콘은 최소로 사용하고 이미지와 텍스트를 이용해 명시적으로 표현하고자 하였다. 

이번 프로젝트를 하면서 공을 많이 들이기도 했고 흥미를 느꼈던 파트라 디자이너의 관점에서 서비스를 이해하고 이를 나름대로 표현해볼 수 있는 기회였다.
